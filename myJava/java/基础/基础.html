&和&&

& 短路与 逻辑与 两种功能 逻辑与就是二进制的与运算
短路与就是 两边都为真时，结果才为真

&& 也是短路与，但是先判断左边，若左边为false，则不会执行右边的语句

左移 右移

右移一位代表除以2 >>
左移一位代表乘以2 <<

i++

int i  = 1;
i = i++;
System.out.println(i); java
很神奇的结果为1
对于 JVM 而言，它对自增运算的处理，是会先定义一个临时变量来接收 i 的值，然后进行自增运算，最后又将临时变量赋给了值为 2 的 i，所以最后的结果为 1。
类似于这样
int i = 1;
int temp = i;
i++;
i = temp;
System.out.println(i);

面向对象

与面向过程的区别

面向过程是以过程为核心，通过函数完成任务
面向对象是以对象为核心，通过对象交互完成任务，程序结构是类和对象组成的模块化结构，代码可以通过继承、组合、多态等方式复用

封装继承多态

封装就是 某变量只能函数内部访问
多态：函数名相同，参数不同

抽象类和接口

抽象类符合 is-a 的关系，而接口更像是 has-a 的关系
类只能继承一个，但是接口可以继承多个
抽象类可以有构造方法，抽象方法(没有实现)，具体方法(代码实现了)。
接口只能包含常量和抽象方法
  1. 抽象类可以有普通方法和抽象方法，接口只能有普通方法。
  2. 抽象类只能继承一个抽象类，接口可以继承多个接口
  3. 抽象类可以有自己的属性，接口只能有常量

成员变量和局部变量

  1. 从语法形式上看：成员变量是属于类的，⽽局部变量是在⽅法中定义的变量或是⽅法的参数；成员变量可以被 public , private , static 等修饰符所修饰，⽽局部变量不能被访问控制修饰符及 static 所修饰；但是，成员变量和局部变量都能被 final 所修饰。
  2. 从变量在内存中的存储⽅式来看：如果成员变量是使⽤ static 修饰的，那么这个成员变量是属于类的，如果没有使⽤ static 修饰，这个成员变量是属于实例的。对象存于堆内存，如果局部变量类型为基本数据类型，那么存储在栈内存，如果为引⽤数据类型，那存放的是指向堆内存对象的引⽤或者是指向常量池中的地址。
  3. 从变量在内存中的⽣存时间上看：成员变量是对象的⼀部分，它随着对象的创建⽽存在，⽽局部变量随着⽅法的调⽤⽽⾃动消失。
  4. 成员变量如果没有被赋初值：则会⾃动以类型的默认值⽽赋值（⼀种情况例外:被 final 修饰的成员变量也必须显式地赋值），⽽局部变量则不会⾃动赋值。

final的作用

  1. 修饰类时，表示这个类不能被继承
  2. 修饰方法，表示这个方法不能被重写
  3. 修饰变量，表示这个变量不能被改变，如果存的是地址，地址指向的区域的内容可以改变。

形近词

finally 是 Java 中异常处理的一部分，用来创建 try 块后面的 finally 块。无论 try 块中的代码是否抛出异常，finally 块中的代码总是会被执行。通常，finally 块被用来释放资源，如关闭文件、数据库连接等。
finalize 是Object 类的一个方法，用于在垃圾回收器将对象从内存中清除出去之前做一些必要的清理工作。不能主动调用，垃圾回收器会自动调用。

创建对象有哪几种方式

1.new创建
  Person person = new Person();
2.反射创建
  Class clazz = Class.forName("Person");
  Person person = (Person) clazz.newInstance();
3.clone创建
  Person person = new Person();
  Person person2 = (Person) person.clone();
4.序列化机制创建
  Person person = new Person();
  ObjectOutputStream oos = new ObjectOutputStream(new FileOutputStream("person.txt"));
  oos.writeObject(person);
  ObjectInputStream ois = new ObjectInputStream(new FileInputStream("person.txt"));
  Person person2 = (Person) ois.readObject();
new对象时父类子类构造方法的执行顺序：
  1. 首先执行父类的静态代码块（仅在类第一次加载时执行）。
  2. 接着执行子类的静态代码块（仅在类第一次加载时执行）。
  3. 再执行父类的构造方法。
  4. 最后执行子类的构造方法。

使用工具类Arrays.asList把数组转换成集合时，不能使用其修改集合相关的方法，它的add /remove/clear方法会抛出UnsupportedOperationException异常。
说明：asList的返回对象是一个Arrays内部类，并没有实现集合的修改方法。Arrays.asList体现的是适配器模式，只是转换接口，后台的数据仍是数组。Stringstr=newString{yang”.'guan",“bao”};
Listlist=Arrays.asList(str); 第一种情况：list.add（yangguanbao"）;运行时异常。 第二种情况：str[0]=“change"；list中的元素也会随之修改，反之亦然。

总结：这个方法创建的list就不要想着增加或者删除元素了(源码没有这个方法)，但是可以修改

异常处理

编译时异常（Checked Exception）：这类异常在编译时必须被显式处理（捕获或声明抛出）。
如果方法可能抛出某种编译时异常，但没有捕获它（try-catch）或没有在方法声明中用 throws 子句声明它，那么编译将不会通过。例如：IOException、SQLException 等。
运行时异常（Runtime Exception）：这类异常在运行时抛出，它们都是 RuntimeException 的子类。对于运行时异常，Java 编译器不要求必须处理它们（即不需要捕获也不需要声明抛出）。
运行时异常通常是由程序逻辑错误导致的，如 NullPointerException、IndexOutOfBoundsException 等。

抛异常
try {
    //包含可能会出现异常的代码以及声明异常的方法
}catch(Exception e) {
    //捕获异常并进行处理
}finally {
    //可选，必执行的代码
}

如果catch到了异常，但是finally里面也抛出了异常
那么最终返回的就是finally的异常，catch的异常会被覆盖。
