工厂模式 - wolai 笔记
工厂模式最大目的是用来解耦
环设计背景：咖啡店，产品有拿铁和美式咖啡

开闭原则：扩展开放，对修改关闭。即在程序扩展的时候，不能修改原有的代码，实现热插拔的效果
工厂模式的优点就是解耦

简单工厂模式

更像是一种编程思维，并不是一种设计模式
根据传入参数不同，返回不同的对象
由于不同的产品都继承了Coffee，所以返回的是Coffee
这样做虽然解决了返回咖啡的耦合，但是产生了新的耦合
如果添加了新的咖啡，就要修改工厂类的代码，违反了开闭原则
所以简单工厂只是一种思维

工厂方法模式

解决了上述模式的缺点

主要角色

  • 抽象工厂(Abstract Factory): 提供了创建产品的接口，调用者通过它访问具体工厂的工厂方法来创建产品。
  • 具体工厂(ConcreteFactory): 主要是实现抽象工厂中的抽象方法，完成具体产品的创建。
  • 抽象产品(Product): 定义了产品的规范，描述了产品的主要特性和功能。
  • 具体产品(ConcreteProduct): 实现了抽象产品角色所定义的接口，由具体工厂来创建，它同具体工厂之间一一对应。

内容

  1. 每个产品对应一个工厂，工厂的返回值就是对应的产品
public class LatteCoffeeFactory implements CoffeeFactory {
  public Coffee createCoffee() {
    return new LatteCoffee();
  }
}

public class AmericanCoffeeFactory implements CoffeeFactory {
  public Coffee createCoffee() {
    return new AmericanCoffee();
  }
}
  1. 咖啡店下单的时候，传的参数也是工厂
public class CoffeeStore {
  public static void main(String[] args){
    //可以根据不同的工厂，创建不同的产品!!!!!!
    CoffeeStore coffeeStore = new CoffeeStore(new LatteCoffeeFactory());
    Coffee latte = coffeeStore.orderCoffee();
    System.out.println(latte.getName());
  }
  private CoffeeFactory factory;
  
  public CoffeeStore(CoffeeFactory factory) {
    this.factory = factory;
  }
  
  public Coffee orderCoffee(String type) {
    Coffee coffee = factory.createCoffee();
    coffee.addMilk();
    coffee.addsugar();
    return coffee;
  }
}
这样就实现了工厂和产品之间的解耦
添加新的产品时，只需要创建一个新的工厂类即可，不需要修改原有工厂的代码
缺点：每新增一个产品，就要新增一个工厂类

抽象工厂模式

工厂方法模式只考虑生成同等级产品，抽象工厂可以处理多等级产品的生产

抽象工厂模式是工厂方法模式的升级版本，工厂方法模式只生产一个等级的产品，而抽象工厂模式可生产多个等级的产品。 一个超级工厂创建其他工厂，该超级工厂又称为其他工厂的工厂。
